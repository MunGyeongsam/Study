### 세션 1: 4B.2 Tutorial Hints 구현

#### 설계 결정
- **방향**: 첫 플레이 감지(`totalRuns == 0`) → 상황별 미니멀 힌트 순차 표시
- **B안 + 주목 연출 (하이브리드)**: PlayScene 내부 HUD 레이어 + 슬로모/딤/글리치 텍스트
- **순수 텍스트만 띄우는 B안 기각**: 탄막 게임 중 텍스트만으로는 시선 확보 불가
- **A안(Scene 오버레이) 기각**: transparent Scene에서 입력 전달 복잡 + 과잉 설계

#### 연출 설계
- Phase machine: `delay → entering → showing → completing → delay → ...`
- 슬로모 진입 (timeScale 0.15) → 화면 딤 (alpha 0.35) → 글리치 텍스트 타이핑
- 액션 성공 시 → 글리치 아웃 + 속도 복귀 (1.0)
- 기존 연출 재활용: gameState.timeScale (포커스 슬로모), 글리치 텍스트 (보스 처치), 딤 오버레이 (일시정지)

#### 힌트 시퀀스 (4단계)
| # | 트리거 | 힌트 | 완료 조건 |
|---|--------|------|----------|
| 1 | 게임 시작 0.5초 후 | `W A S D -- MOVE` | 이동 입력 감지 |
| 2 | Move 완료 1초 후 | `SHIFT -- DASH` | 대쉬 사용 (cooldownTimer > 0) |
| 3 | Dash 완료 1초 후 | `SPACE -- FOCUS (SLOW TIME)` | 포커스 활성 (focus.active) |
| 4 | Focus 완료 1초 후 | `WEAPON FIRES AUTOMATICALLY` | 3초 타이머 |

#### timeScale 충돌 해결
- tutorialHints.update()는 ecsManager.update() **이후** 실행
- 포커스 시스템이 timeScale을 설정해도 tutorial이 오버라이드
- 완료(completing) phase에서 timeScale → 1.0 복귀 시 포커스가 자연스럽게 인수
