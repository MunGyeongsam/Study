### 세션 1: 5A.3f 계획 수립

#### 설계 결정

**목표**: 변형 4종 + 포메이션 5종이 실제 스테이지에서 의미 있게 등장하도록 규칙 시스템 구축

**접근법 비교**:
| 선택지 | 내용 | 판정 |
|--------|------|------|
| A. 전부 손작업 | Stage 1~15 개별 데이터 정의 | ❌ 데이터 물량 = 제약의 미학 위반 |
| B. 규칙만 | 글로벌 규칙 강화 | ❌ "특별한 순간" 보장 안 됨 |
| **C. 하이브리드 (규칙 중심)** | 핵심 순간 보장 + 나머지 규칙 | ✅ 3줄 규칙 = 15개 테이블 대체 |

**가치관 검증**:
- **재미**: ✅ 4기둥 모두 기여 (긴장감+놀라움+성취감+몰입감)
- **쾌감**: ✅ 변형 "첫 만남" 보장 → 놀라움 극대화
- **제약의 미학**: ✅ 코드 규칙으로 무한 스테이지 다양성 자동 생성

#### 4가지 규칙 설계

1. **스테이지 테마**: 기동형(bit/vector)/화력형(node/loop/matrix) 교대 → 스테이지마다 다른 느낌
2. **변형 보장 등장**: Stage 4/7/10/13 Wave 1에 해당 변형 1체 확정 스폰
3. **변형 확률 스케일링**: baseChance × (1 + (stage-first) × 0.03), cap 25%/전체 50%
4. **포메이션 + 변형 조합**: 포메이션 내 적에게도 변형 적용
