  - `CURVE_RENDER_LOGGED_OK/MISSING`를 entity ID가 아닌 renderable 인스턴스 키로 변경
  - weak table (`__mode = "k"`) 적용

#### 렌더 실험 (시행착오)
- `drawScale = 0.01` + `polygon("fill")` + 정규화 비활성 → 너무 작거나 뭉개짐
- 다양한 scale/mode 조합 시도 → Gallery/Play 일관성 확보 실패
- **결론**: 정점 수동 변환 방식 자체가 문제 → 세션 7에서 근본 해결

#### 추가 기능
- `stageManager.lua`: Force DNA Spawn 모드 (F키), `DNA_ROLL_LOG_ENABLED` 플래그
- `playScene.lua`: `[`/`]` 또는 `9`/`0` 커브 두께 조절, Force DNA 뱃지 + 토스트

### 세션 7: 커브 오버레이 렌더 정상화 (transform stack + lineWidth 보상)

#### 문제 진단
- `_drawCurveOverlay`가 매 프레임 모든 정점에 offset/scale 산술 + 새 table 할당
- `_drawBodyLayers`는 `lg.push/translate/scale/pop`으로 GPU 변환 활용 → 비교에서 차이 발견

#### 해결 (3단계)

**1단계: 정규화 캐시 분리**
- `NORMALIZED_CURVE_CACHE` 추가 — center offset + scaleToUnitRadius 적용한 점을 한 번만 계산
- `_getNormalizedCurvePoints(curveName, norm)` 함수 추가
- 프레임당 테이블 할당 0, 정점 산술 0

**2단계: lg.push/translate/scale 방식 전환**
- `_drawCurveOverlay`: `lg.push() → translate(x,y) → scale(r) → line(cachedVerts) → pop()`
- `galleryScene.drawCurveOverlay`: 동일 패턴 적용

**3단계: lineWidth 보상 (카메라 스케일)**
- 문제: `lg.scale(r)` 후 lineWidth도 r배 확대 → Gallery(radius=25)에서 선 25px
- 1차 시도: `lineWidth / r` → Gallery 정상, Play에서 카메라 96배 확대로 선 폭발
- 최종: `inverseTransformPoint`로 "1px = 월드 몇 단위" 계산
  - `_pxInWorld = abs(inverseTransformPoint(1,0) - inverseTransformPoint(0,0))`
  - lineWidth: `CURVE_OVERLAY_STYLE.lineWidth * _pxInWorld / r`
  - Gallery: 카메라 없으므로 `_pxInWorld=1` → `1/radius` → 1px
  - Play: `_pxInWorld≈0.01` → `0.01/0.3` → 카메라 확대 후 ~1px
