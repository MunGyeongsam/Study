> 최종 수정: 2026-04-21
> 상태: **5종 구현 완료** / Worm 미구현 / 확장(변형+포메이션) 설계 추가

---

## 목표

기존 적 4종(basic, spiral, aimed, wave)을 **세계관 기반 6종**으로 교체하여:
- 시각적 정체성 부여 (벡터 아트 렌더링)
- 회피 방식의 다양성 확보 (접촉형, 포탑형, 돌진형, 격자형 등)
- 스테이지 진행에 따른 **전략적 조합** 가능

## 적 6종 상세 설계

### 1. Bit — 접촉형 무리

> "최소 단위 오염 데이터. 혼자는 약하지만, 무리지으면 공간을 삼킨다."

| 항목 | 설계값 | 실제 구현 |
|------|--------|----------|
| 형태 | 작은 점/원 (반지름 0.08) | circle (0.08) ✅ |
| 색상 | 흰색 `{1.0, 1.0, 1.0}` | ✅ |
| HP | 1 | ✅ |
| XP | 1 | ✅ |
| AI | **swarm** — 플레이어 방향 직선 돌진 | swarm (0.8) ✅ |
| 탄막 | **없음** — 몸통 접촉만 | ✅ |
| 이속 | 0.8 | ✅ |
| 스폰 | 한 번에 5~8마리 무리 | **3~5마리** (축소 조정) |

**새 AI 행동: `swarm`**
- 플레이어 방향으로 직선 이동 (chase와 유사하지만 정지 없음)
- 무리 스폰 시 약간의 위치 분산 (반경 0.3 랜덤)
- 벽/경계 도달 시 LifeSpan으로 자연 소멸

**재미 포인트:** 탄막은 안 쏘지만 "몸 자체가 탄막". 여러 적과 동시에 나오면 탄막 피하면서 Bit 무리도 관리해야 함.

### 2. Node — 정지 포탑

> "오염된 데이터 노드. 공간에 뿌리내려 움직이지 않지만, 끊임없이 신호를 뿜는다."

| 항목 | 값 |
|------|-----|
| 형태 | 사각형 (회전하는 diamond) |
| 색상 | 빨강 `{1.0, 0.3, 0.2}` |
| HP | 4 |
| XP | 3 |
| AI | **stationary** — 스폰 위치에 고정, 회전만 | ✅ |
| 탄막 | `circle` (8발, 속도 1.0, rate 1.0) | **ring_pulse** (8발, 속도 1.2, rate 1.2) |
| 히트박스 | 0.15 | ✅ |

**새 AI 행동: `stationary`**
- 이동 없음. 스폰 위치에 고정
- 렌더링: 천천히 회전하는 다이아몬드 (visual만)

**재미 포인트:** "죽이러 가야 하는 적". 탄막을 뚫고 접근해서 처리 vs 무시하고 피하기 — 선택 발생.

### 3. Vector — 고속 직선 돌진

> "방향성 유해 프로세스. 경고 한 번, 그리고 관통."

| 항목 | 값 |
|------|-----|
| 형태 | 화살표 (삼각형 + 꼬리선) |
| 색상 | 노랑 `{1.0, 0.9, 0.1}` |
| HP | 2 |
| XP | 4 |
| AI | **charge** (신규) — 짧은 경고 후 고속 직선 돌진 |
| 탄막 | **없음** — 몸통 돌진이 위협 |
| 히트박스 | 0.12 |

**새 AI 행동: `charge`**
1. **경고 단계** (0.8초): 스폰 위치에서 플레이어 방향으로 조준, 깜빡임
2. **돌진 단계**: 조준 방향으로 고속 직선 이동 (속도 4.0) — 방향 고정, 추적 안 함
3. **소멸**: 화면 밖 도달 또는 LifeSpan 만료

**재미 포인트:** Bit와 대비 — Bit는 느리고 많고, Vector는 빠르고 하나. 경고→회피 타이밍이 핵심.

### 4. Loop — 원형 궤도 나선

> "무한 반복 프로세스. 끝나지 않는 순환 속에서 데이터를 오염시킨다."

| 항목 | 값 |
|------|-----|
| 형태 | 나선/원 (동심원 2겹) |
| 색상 | 보라 `{0.8, 0.3, 1.0}` |
| HP | 5 |
| XP | 5 |
| AI | `orbit` (기존) — 스폰 중심 공전 |
| 탄막 | `spiral` (4발, 속도 1.0, turn 2.0) |
| 히트박스 | 0.18 |

**기존 spiral과 차이:** 렌더링만 변경 (원→나선 형태). 이미 재미 검증된 행동.

### 5. Matrix — 격자 탄막

> "오염된 데이터 구조체. 질서정연한 격자로 공간을 지배한다."

| 항목 | 값 |
|------|-----|
| 형태 | 육각형 / 격자 다각형 |
| 색상 | 시안 `{0.2, 1.0, 0.8}` |
| HP | 6 |
| XP | 6 |
| AI | `drift` (기존) — 느린 하강 |
| 탄막 | **grid** (신규) — 십자/격자형 탄막 |
| 히트박스 | 0.16 |

**탄막 패턴: `cross`** (설계 시 `grid`에서 축소)
- 상하좌우 4방향 동시 발사 (대각 4방향은 미구현)
- 일정 간격으로 반복 → 십자 형태의 탄막
- 설계의 8방향 grid는 향후 확장 가능

**재미 포인트:** 기존 원형/나선 탄막과 완전히 다른 회피법. 격자 틈새를 읽어야 함.

### 6. Worm — 뱀 이동 (후순위)

> "자기 복제 연쇄. 머리를 잡아도 몸통이 남는다."

| 항목 | 값 |
|------|-----|
| 형태 | 연결된 원 체인 (5~7 세그먼트) |
| 색상 | 초록 `{0.3, 1.0, 0.4}` |
| HP | 머리 8 (세그먼트는 무적) |
| XP | 8 |
| AI | **snake** (신규) — 사인파 이동, 몸통 추종 |
| 탄막 | 없음 (몸통 자체가 장애물) |

**구현 복잡도:** 높음 — 다관절 위치 히스토리, 세그먼트별 렌더링, 히트박스 체인
→ **별도 태스크로 분리. 3B 기본 5종 완료 후 검토.**

# Phase 2: 적 다양성 확장 — 변형(Variant) + 포메이션(Formation)

## 확장 목표

기존 5종의 기반 위에 **변형 시스템**과 **포메이션 시스템**을 추가하여:
- 실질 적 종류: 5종 → 5×5 = **25종 조합** (기본 + 변형 4종)
- 스폰 패턴: 랜덤 위치 → **전술적 대형 5종**
- 스테이지 확장 소재 / Endless 모드 무한 스케일링 기반

## 재미 검증

| 기둥 | 질문 | 검증 |
|------|------|------|
| **도전** | 같은 적인데 더 어려운가? | ✅ Swift-Bit 무리 = 반응 속도 한계, Shielded-Node = 우회 강제 |
| **학습** | 변형 구별→대응법 숙달? | ✅ 외형 힌트(잔상/두꺼운선/점선/아크)로 즉시 인지 가능 |
| **선택** | "어떤 변형부터?" 판단? | ✅ Splitter는 나중에, Armored는 무시 등 전략 분기 |
| **놀라움** | 포메이션이 놀라운가? | ✅ V자 Bit 편대, 양쪽 Vector 차지 = 첫 대면 임팩트 |

## 변형(Variant) 시스템

### 설계 원칙
- 기존 적의 **스탯 배수 + 외형 오버레이**만으로 구현
- 새 AI 행동 불필요 (Shielded만 피격 방향 체크 추가)
- 모든 변형은 **모든 기본 적과 조합 가능** (5×4 = 20 조합)

### 변형 4종 상세

#### 1. Swift — 고속 소형

| 항목 | 값 |
|------|-----|
| 스탯 변경 | 속도 ×1.5, 크기 ×0.8, HP ×0.8 |
| 외형 힌트 | 잔상(ghost trail) 2~3개 — 이전 프레임 위치에 반투명 도형 |
| XP 보너스 | ×1.2 |
| 첫 등장 | Stage 4 |

**조합 하이라이트:**
- Swift-Bit: 빠른 무리 돌진 → 긴장감 대폭 상승
- Swift-Vector: 경고 시간 동일하지만 돌진 속도 6.0 → 반응 극한

#### 2. Armored — 고체력 대형

| 항목 | 값 |
|------|-----|
| 스탯 변경 | HP ×2.5, 속도 ×0.7, 크기 ×1.3 |
| 외형 힌트 | 두꺼운 외곽선(stroke width ×3) |
| XP 보너스 | ×1.5 |
| 첫 등장 | Stage 7 |

**조합 하이라이트:**
- Armored-Node: 포탑인데 안 죽음 → "무시하고 피할까, 시간 투자해서 잡을까"
- Armored-Loop: 궤도 탱크 — 나선탄 오래 지속

#### 3. Splitter — 분열형

| 항목 | 값 |
|------|-----|
| 스탯 변경 | HP ×1.0 (동일), 사망 시 미니 2마리 분열 |
| 분열체 | 원본의 크기 ×0.5, HP 1, XP 0, 탄막 없음, AI 동일 |
| 외형 힌트 | 점선 외곽(dashed outline) |
| XP 보너스 | ×1.3 |
| 첫 등장 | Stage 10 |

**조합 하이라이트:**
- Splitter-Bit: 무리가 잡으면 더 불어남 → 우선순위 전략 역전
- Splitter-Loop: 궤도 적 잡으면 미니 2마리 → 나선탄 + 분열체 동시 처리

#### 4. Shielded — 전방 방어

| 항목 | 값 |
|------|-----|
| 스탯 변경 | HP ×1.0 (동일), 전방 90° 탄막 무효 |
| 방패 판정 | 적 진행 방향 기준 ±45° 내 피탄 무효 |
| 외형 힌트 | 반원 방패 아크 (진행 방향쪽) |
| XP 보너스 | ×1.5 |
| 첫 등장 | Stage 13 |

**조합 하이라이트:**
- Shielded-Matrix: 전방 방어 + 십자탄 → 우회 기동 필수
- Shielded-Vector: 정면 차지를 못 막음 → 측면에서 잡아야

## 포메이션(Formation) 시스템

### 설계 원칙
- stageManager의 `_spawnWave()`에서 **좌표 배열만 계산**하여 기존 `createEnemy()` 호출
- 새 컴포넌트/시스템 불필요
- 포메이션 난이도 등급(1~3)으로 스테이지 곡선 반영

### 포메이션 5종 상세

| # | 이름 | 배치 | 적합 타입 | 난이도 | 재미 |
|---|------|------|----------|:------:|------|
| 1 | **V자 편대 (Wedge)** | 중앙 리더 + 좌우 대칭 날개 | Bit ×7~9 | 1 | "탄막벽이 밀려온다" |
| 2 | **포위 (Pincer)** | 좌+우 동시 스폰 | Vector ×2 | 2 | 양쪽 차지 → 도망칠 곳 읽기 |
| 3 | **삼각 포탑 (Triangle)** | 삼각형 꼭짓점 3곳 | Node ×3 | 2 | 탄막 교차 지대 → 안전지대 찾기 |
| 4 | **호위 (Escort)** | 중앙 탄막적 + 주변 Bit 4~6 | Mixed | 3 | 탄막적 보호하는 무리 → 우선순위 판단 |
| 5 | **나선 행렬 (Spiral Array)** | 원호를 따라 등간격 | Loop ×3~4 | 3 | 겹치는 나선탄 → 패턴 읽기 |

### 포메이션 좌표 계산

```lua
-- Wedge (V자): 리더 중앙, 좌우로 offset
for i = 1, count do
  local row = math.ceil(i / 2)
  local side = (i % 2 == 1) and 1 or -1
  offsets[i] = { x = side * row * spacing, y = row * spacing }
end

-- Pincer: 좌측 N마리 + 우측 N마리
-- Triangle: 정삼각형 꼭짓점 (60° 간격)
-- Escort: 중앙 1 + 원형 배치 N마리
-- Spiral Array: 원호 등분
```
