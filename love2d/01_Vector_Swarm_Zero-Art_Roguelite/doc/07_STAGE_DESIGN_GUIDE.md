# 07. 스테이지 디자인 가이드

> 스테이지 진행 시스템의 설계 방향과 확장 계획을 정리하는 문서.
> 마지막 갱신: 2026-04-19

---

## 1. 설계 철학

### 핵심 원칙
- **점진적 긴장**: 스테이지가 올라갈수록 자연스럽게 위험도 증가
- **무한 확장**: 정의된 스테이지 이후는 공식으로 자동 생성 (데이터 없어도 동작)
- **데이터 드리븐**: 스테이지 특성은 코드 수정 없이 테이블 편집만으로 조정
- **확장 슬롯**: 나중에 보스, 특수 조건, 환경 변화 등 필드 추가만 하면 됨

### 현재 구조 (Phase 2D + 3A)
```
Stage N → Wave 1..M (적 전멸 클리어) → "STAGE CLEAR!" → Stage N+1 → ...
Stage 3,6,9,12,15 → 보스 스테이지 (보스 1체 처치 = 클리어)
Stage 16+ → 매 3스테이지마다 보스 반복 (무한 스테이지)
```

### 미래 구조 (Phase 3+)
```
Stage 1 [존 A: 3웨이브] → 이동 → Stage 2 [존 B: 4웨이브] → ... → [보스 스테이지]
```

## 2. 스테이지 정의 테이블

### 수동 디자인 스테이지 (1~5)

| Stage | Waves | 스폰 방향 | 적 타입 풀 | 의도 |
|-------|-------|----------|-----------|------|
| 1 | 3 | 위 100% | basic | 입문. 위만 보면 OK |
| 2 | 4 | 위 80%, 좌우 각 10% | basic, aimed | 좌우 인지 시작 |
| 3 | 5 | 위 60%, 좌우 각 15%, 아래 10% | basic, spiral, aimed | 사방 경계 본격화 |
| 4 | 5 | 위 50%, 좌우 각 20%, 아래 10% | 전체 4종 | 풀 라인업 등장 |
| 5 | 6 | 위 40%, 좌우 각 20%, 아래 20% | 전체 4종 | 고난이도 진입 |

### 자동 생성 스테이지 (6+)

Stage N (N ≥ 6) 일 때:
- **Waves**: `min(5 + floor((N-5)/2), 8)` → 최대 8웨이브
- **스폰 방향**: 위 40%, 좌우 각 20%, 아래 20% (Stage 5와 동일, 고정)
- **적 타입**: 전체 4종 사용
- **난이도 수식**: 기존 공식 유지 (cap 적용)

## 3. 난이도 스케일링

### 파라미터 공식

| 파라미터 | 공식 | Cap |
|----------|------|-----|
| 웨이브당 적 수 (기본) | `3 + (stage-1) * 2` | 15 |
| 적 HP 배율 | `1.0 + (stage-1) * 0.15` | 없음 (무한 성장) |
| 적 이동속도 배율 | `1.0 + (stage-1) * 0.1` | 2.0x |
| 탄막 속도 배율 | `1.0 + (stage-1) * 0.05` | 1.5x |
| 웨이브 간격 | `max(1.5, 3.0 - (stage-1) * 0.3)` | 최소 1.5초 |

### 스테이지 내 웨이브 램프
- 웨이브 1: 기본 적 수의 ~50%
- 웨이브 중반: ~75%
- 최종 웨이브: 100%
- 공식: `count * (0.5 + 0.5 * wave/wavesPerStage)`

## 4. 스폰 위치 규칙

### 방향별 스폰 위치
```
          ┌─────────────────┐
          │  TOP spawn zone │  Y = playerY + 6 + rand(0~2)
          │                 │
     LEFT │                 │ RIGHT
     X=-3.5                 │  X=+3.5
     ±rand│   camera view   │  ±rand
          │   (~4.5 x 10)   │
          │                 │
          │                 │
          │  BTM spawn zone │  Y = playerY - 6 - rand(0~2)
          └─────────────────┘
```

### 안전 규칙
- 모든 스폰은 **카메라 밖 1유닛 이상** 떨어진 곳
- 월드 경계(±10, ±15) 내로 clamp
- 아래 스폰 적: drift AI 사용 금지 (chase/orbit만) — 아래로 도망 방지

## 5. 확장 계획

### Phase 3A: 보스 스테이지 (구현 완료 ✅)

`stageManager`에서 `isBossStage` 플래그로 보스 스테이지를 감지하며, 웨이브 대신 보스 1체만 등장:

| 보스 | 스테이지 | HP | Phase | 특징 |
|------|:---:|:---:|:---:|------|
| NULL | 3 | 50 | 2 | 기본 탄막 (튜토리얼) |
| STACK | 6 | 100 | 3 | circle + aimed 조합 |
| HEAP | 9 | 160 | 3 | 텔레포트 + 점멸 경고 |
| RECURSION | 12 | 200 | 3 | 공전 이동 + 미니언 소환 |
| OVERFLOW | 15 | 300 | 4 | 전 패턴 순환 + Phase별 색상/이동 |

보스 처치 보상: 탄막 전소거 + XP 폭발 + HP/대쉬/포커스 회복

### Phase 3B+: 존 여정 & 환경 변화 (예정)
```lua
-- 확장 필드 예시
STAGE_DEFS[10] = {
    waves = 1,
    clearType = "boss",        -- "kill_all" | "boss" | "survive"
    boss = "NULL",             -- 보스 ID
    arenaSize = {25, 35},      -- 스테이지별 아레나 크기
    bgColor = {0.05, 0, 0.1},  -- 배경색 변경
    music = "boss_theme",      -- BGM 전환
}
```

### Phase 4: 메타 진행
- 스테이지 클리어 기록 → 영구 해금 (Data Fragment)
- 특정 스테이지 도달 → 신규 무기/캐릭터 해금
- "이번 런에서 Stage N 최초 도달" 실적

### 적 타입 확장 (Phase 3B)
```lua
-- 새 적 타입은 STAGE_DEFS.types에 추가만 하면 됨
[7] = { waves = 5, types = {"basic","spiral","aimed","wave","matrix","worm"} },
```

## 6. 밸런스 레퍼런스

### 세션 페이스 목표
| 구간 | 시간 | 스테이지 |
|------|------|---------|
| 입문 | 0~2분 | Stage 1~2 |
| 중반 | 2~6분 | Stage 3~5 |
| 후반 | 6~10분 | Stage 6~8 |
| 하드코어 | 10~15분 | Stage 9+ |

### 레드라인 (절대 규칙)
- Stage 1~2: 즉사 금지
- 1초 이내 화면 가득 탄막 금지
- 최소 2가지 회피 경로 존재
- 적 15초 생존 → 자동 디스폰 (병목 방지)
