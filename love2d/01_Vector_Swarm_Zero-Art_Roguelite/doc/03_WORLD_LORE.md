# 03. 세계관 (World Lore)

> Vector Swarm: Zero-Art Roguelite — 세계관 및 서사 설계

---

## 3.1. 세계 개요

### 3.1.1. 배경: 데이터스피어 (Datasphere)

모든 디지털 기기의 깊은 곳, 전자 신호가 흐르는 영역 너머에 **데이터스피어**라 불리는 추상적 공간이 존재한다.

데이터스피어는 데이터가 태어나고, 흐르고, 소멸하는 공간이다. 이곳의 모든 것은 **벡터(Vector)** — 방향과 크기를 가진 순수한 수학적 존재 — 로 이루어져 있다. 색은 전자의 파장이고, 형태는 방정식이며, 소리는 주파수의 진동이다.

### 3.1.2. 대재앙: 스웜 오버플로우 (Swarm Overflow)

데이터스피어는 오랫동안 안정적이었다. 그러나 어느 순간, 알 수 없는 공명(resonance)이 발생한다. 잠들어 있던 데이터 파편들이 깨어나 스스로 복제하기 시작했다. 이들은 **스웜(Swarm)** — 방향 없이 증식하는 유해 벡터의 군집 — 이 되어 데이터스피어 전역을 침식하기 시작한다.

스웜은 점점 복잡한 패턴을 형성하며, 자신을 방어하기 위해 **탄막(Danmaku)** — 수학 함수로 정의된 공격 패턴 — 을 생성한다. 아름답지만 치명적인 기하학적 탄막이 데이터스피어를 가득 채운다.

### 3.1.3. 존재하는 세계의 형태

데이터스피어에는 물리적 실체가 없다. 모든 것은:
- **선(Line)** 으로 경계가 그려지고
- **점(Point)** 으로 위치가 정의되고
- **원(Circle)** 과 **다각형(Polygon)** 으로 존재가 표현되며
- **빛(Glow)** 으로 에너지가 가시화된다

이것이 곧 **Zero-Art** — 이 세계에 "그림"이란 없다. 오직 수학과 빛만이 존재한다.

## 3.2. 플레이어: 커서 (The Cursor)

### 3.2.1. 정체

플레이어는 **커서(Cursor)** — 데이터스피어에 파견된 정화 프로세스다.

커서는 스웜을 제거하고 오염된 데이터를 복구하기 위해 생성된 단일 벡터 개체다. 작지만, 데이터스피어의 근본 법칙을 조작할 수 있는 권한을 가지고 있다.

### 3.2.2. 능력

| 능력 | 설명 | 세계관적 근거 |
|------|------|--------------|
| 이동 | 데이터 흐름을 타고 자유롭게 이동 | 벡터로서의 방향성 |
| 대쉬 | 순간적으로 좌표를 건너뜀 | 포인터 점프 (메모리 주소 이동) |
| 포커스 | 주변 시간 흐름을 느리게 함 | 프로세스 우선순위 상승 (CPU 독점) |
| 공격 | 정화 신호를 발사하여 스웜을 해체 | 가비지 컬렉션 트리거 |
| 레벨업 | 스웜 해체 시 남는 잔여 데이터를 흡수하여 성장 | 메모리 재할당 |

### 3.2.3. 원시 패턴의 축복: 신(Deity)

스웜 오버플로우 이전부터 데이터스피어에는 순수한 수학 법칙으로 이루어진 **원시 패턴(Primal Patterns)** 이 존재했다. 이들은 의식을 가진 존재가 아니지만, 커서와 공명하여 자신의 힘을 빌려줄 수 있다.

커서가 데이터스피어에 진입할 때, 4개의 원시 패턴이 동시에 공명을 보내온다. 커서는 그 중 하나를 선택하여 자신의 코어에 각인한다 — 이것이 **의식(Ritual)**.

| 신 | 곡선 | 정체 | 축복 |
|----|------|------|------|
| **Rose** | Rose Curve | 꽃잎 구조의 재생 패턴 | 스침 시 데이터 복원 (HP 회복) |
| **Cycloid** | Epicycloid | 회전 기어의 파괴 패턴 | 타격에 임계점 부여 (크리티컬) |
| **Lemniscate** | Lemniscate | 무한 루프의 순환 패턴 | 행동의 되감기 (쿨다운 리셋) |
| **Inferno** | Devil's Curve | 악마 곡선의 폭발 패턴 | 소멸 에너지의 연쇄 방출 (AOE) |

> 상세 설계: `doc/20_DEITY_SYSTEM_DESIGN.md` 참조

### 3.2.4. 죽음과 재시작 (로그라이트 루프의 근거)

커서가 스웜에 의해 파괴되면, 데이터스피어의 **체크포인트 프로토콜**이 작동한다. 커서는 원점(Origin)에서 재생성된다. 다만, 이전 런에서 획득한 순수 데이터 — **데이터 프래그먼트(Data Fragment)** — 는 체크포인트에 보존되어, 다음 커서의 기반이 된다.

> *"파괴될 때마다 더 강해진다. 이것이 반복의 의미다."*

## 3.3. 적: 스웜 (The Swarm)

### 3.3.1. 스웜의 본질

스웜은 **방향을 잃은 벡터의 집합**이다. 본래 데이터스피어의 일부였으나, 공명에 의해 오염되어 제어 불능의 자기 복제 패턴이 되었다.

### 3.3.2. 스웜 유형

| 유형 | 형태 | 행동 | 세계관 | 탄막 특성 |
|------|------|------|--------|----------|
| **비트(Bit)** | 작은 점/원 | 무리지어 돌진 | 최소 단위 오염 데이터 | 없음 (접촉 피해) |
| **노드(Node)** | 삼각형/사각형 | 위치 고정, 탄막 발사 | 오염된 데이터 노드 | 방사형 기본 패턴 |
| **벡터(Vector)** | 화살표/선분 | 직선 이동, 관통 공격 | 방향성을 가진 유해 프로세스 | 직선/조준탄 |
| **루프(Loop)** | 원/나선 | 원형 궤도, 반복 패턴 | 무한 반복에 빠진 프로세스 | 나선형/회전 패턴 |
| **메트릭스(Matrix)** | 격자/다각형 | 그리드 패턴 | 오염된 데이터 구조체 | 격자형 탄막 |
| **웜(Worm)** | 연결된 선분 체인 | 뱀처럼 이동 | 자기 복제 연쇄 프로세스 | 체인 탄막 |

### 3.3.3. 보스: 코어 바이러스 (Core Virus)

각 구역의 끝에는 **코어 바이러스** — 스웜을 통제하는 상위 프로세스 — 가 존재한다. 이들은 데이터스피어의 근본 함수를 해킹하여 자신만의 복잡한 탄막 패턴을 생성한다.

| 보스 | 구역 | 특징 | 탄막 테마 |
|------|------|------|----------|
| **NULL** | 안전 구역 끝 | 튜토리얼 보스, 단순 패턴 | 원형 방사 |
| **STACK** | 탐험 구역 끝 | 패턴이 층층이 쌓임 | 계단식 + 벽 패턴 |
| **HEAP** | 전투 구역 끝 | 랜덤 폭발, 예측 불가 | 무작위 산개 + 집중탄 |
| **RECURSION** | 위험 구역 끝 | 분열·증식, 작아지며 더 빨라짐 | 프랙탈 패턴 |
| **OVERFLOW** | 최종 보스 | 데이터스피어 공명의 근원 | 모든 패턴 종합 |

## 3.4. 세계 구조 (하단 → 상단 진행)

플레이어는 데이터스피어의 **하층(Origin)** 에서 시작하여, 스웜에 오염된 상층으로 올라가며 정화한다.

```
  ▲ 상단 (y = +125)
  │
  │  ┌─────────────────┐
  │  │  BOSS CHAMBER    │  최종 보스 OVERFLOW
  │  │  (보스 구역)      │
  │  ├─────────────────┤
  │  │  DANGER ZONE     │  고난이도 + 보스 RECURSION
  │  │  (위험 구역)      │
  │  ├─────────────────┤
  │  │  COMBAT ZONE     │  중급 전투 + 보스 HEAP
  │  │  (전투 구역)      │
  │  ├─────────────────┤
  │  │  EXPLORATION     │  초급 탐험 + 보스 STACK
  │  │  (탐험 구역)      │
  │  ├─────────────────┤
  │  │  SAFE START      │  시작 + 튜토리얼 보스 NULL
  │  │  (안전 구역)      │
  │  └─────────────────┘
  │
  ▼ 하단: Origin (0, -100) ← 플레이어 시작점
```

## 3.5. 아이템/성장 요소의 세계관적 해석

| 게임 요소 | 세계관 해석 |
|----------|------------|
| Data Fragment (재화) | 스웜 해체 시 남은 순수 데이터 조각 |
| 무기 | 커서의 정화 알고리즘 변종 |
| 패시브 스킬 | 커서의 실행 환경(런타임) 최적화 |
| 레벨업 | 런타임 메모리 확장 → 새로운 모듈 로드 |
| 영구 강화 | 체크포인트 프로토콜에 축적된 경험 데이터 |
| 대쉬 쿨타임 | 포인터 점프 후 주소 재계산 시간 |
| 포커스 에너지 | CPU 우선순위 독점 가능 잔여량 |

## 3.6. 서사 톤 & 원칙

### 3.6.1. 톤
- **미니멀하고 시적(詩的)인 서사** — 대사가 아닌 환경으로 이야기를 전달
- **직접적 내러티브 최소화** — 컷씬 없음, 텍스트도 최소한
- **간접 서사** — 구역 변화, 배경 패턴, 색상, 음악이 스토리를 전달
- **감성 키워드:** 고독, 반복, 정화, 아름다운 위험

### 3.6.2. 서사 전달 방식
| 방식 | 예시 |
|------|------|
| 구역 진입 시 한 줄 텍스트 | *"깊은 곳에서 무언가가 공명한다..."* |
| 보스 처치 후 한 줄 텍스트 | *"STACK이 해체됐다. 데이터가 안정을 찾는다."* |
| 배경 비주얼 변화 | 안전 구역 → 진한 네이비, 위험 구역 → 붉은 파동 |
| BGM 전환 | 구역별 분위기 변화로 긴장감 전달 |

### 3.6.3. 엔딩 비전
최종 보스 OVERFLOW를 처치하면, 데이터스피어의 공명이 멈추고 세계가 잠시 고요해진다. 하지만 화면 한 구석에서 새로운 파동이 시작된다.

> *"공명은 멈추지 않는다. 다만, 다음 주기가 시작될 뿐."*

→ 2회차(New Game+) / 더 어려운 난이도 루프로 자연스럽게 연결

## 3.7. 세계관 × 재미 연결표

> **"세계관은 재미를 위해 존재한다. 재미에 기여하지 않는 설정은 만들지 않는다."**

### 3.7.1. 세계관 요소가 재미에 기여하는 방법

| 세계관 요소 | 재미 기여 | 재미 기둥 | 구체적 효과 |
|-----------|----------|----------|------------|
| **데이터스피어** (벡터 세계) | Zero-Art의 근거 → 시각적 일관성 | 놀라움 | 무(無)에서 수학만으로 아름다움을 만들어낸다는 설득력 |
| **스웜 오버플로우** (대재앙) | 싸울 이유, 긴장감의 배경 | 긴장감 | "이 세계가 멸망하고 있다" → 전투의 의미 부여 |
| **커서** (플레이어) | 죽음과 재시작의 자연스러운 설명 | 몰입감 | 로그라이트 반복 루프가 세계관적으로 당연함 |
| **체크포인트 프로토콜** (부활) | 사망의 좌절감 완화 | 몰입감 | "죽어도 괜찮다, 더 강해져서 돌아온다" |
| **Data Fragment** (재화) | 성장의 의미 부여 | 성취감 | 단순 숫자가 아닌 "정화한 데이터의 조각" |
| **스웜 유형** (적 분류) | 다양한 적 = 다양한 회피 패턴 | 긴장감+놀라움 | 6종의 적이 매번 다른 전투 경험 |
| **코어 바이러스** (보스) | 보스전의 서사적 무게 | 성취감 | "RECURSION을 처치했다" = 스토리 진전 |
| **구역 진행** (하단→상단) | 난이도 상승의 자연스러운 근거 | 긴장감 | 위로 갈수록 위험한 이유가 세계관에 있음 |
| **간접 서사** (한 줄 텍스트) | 분위기 몰입, 플레이 방해 없음 | 몰입감 | 컷씬으로 Flow를 끊지 않음 |
| **엔딩 루프** ("공명은 멈추지 않는다") | New Game+ 동기 부여 | 몰입감 | "한 번 더" 할 이유를 세계관이 만들어줌 |

### 3.7.2. 세계관 설계 원칙

1. **재미를 설명하는 세계관** — 게임 메커닉이 먼저, 세계관이 그것을 자연스럽게 감싸는 구조
2. **플레이를 방해하지 않는 서사** — 컷씬 없음, 긴 대화 없음, 한 줄 텍스트와 환경으로 전달
3. **죽음이 재미인 구조** — 사망 → 재생성이 세계관적으로 당연하므로 좌절이 아닌 기대
4. **설정은 짧게, 상상은 길게** — 최소한의 설정으로 플레이어의 상상력을 자극
5. **모든 설정은 게임 내에서 느껴야** — 외부 문서를 읽어야만 이해되는 설정은 없앤다
