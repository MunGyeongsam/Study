# 08. Boss System Design — Phase 3A

> **목표:** 보스 시스템 구현으로 스테이지 진행에 정점(climax)을 만든다.  
> **재미 기둥:** 긴장감 + 성취감 + 놀라움 + 몰입감 (4기둥 전부 기여)

---

## 1. 재미 검증 (Fun Gate)

### 4기둥 기여도

| 기둥 | 보스 시스템에서의 표현 |
|------|----------------------|
| **긴장감 (Tension)** | 높은 HP, 복잡한 탄막, 페이즈 전환 — 게임의 최고 긴장 포인트 |
| **성취감 (Mastery)** | 패턴을 읽고 이겨냈다 → 최고의 카타르시스 |
| **놀라움 (Wonder)** | 페이즈마다 다른 공격, 예측 불가한 전개 |
| **몰입감 (Flow)** | 긴장(공격)↔이완(안전구간) 리듬 = 극한 몰입 |

### 검증 체크리스트

- [x] 4기둥 중 최소 1개에 기여? → **4개 전부**
- [x] 즉각적 피드백? → HP바 감소, 페이즈 전환 비주얼, 처치 시 보상 폭발
- [x] 이 기능 없으면 게임이 덜 재미있는가? → YES. 목표 없이 웨이브만 반복 = 루프가 느슨
- [x] 다른 재미 요소를 방해하지 않는가? → NO, 기존 시스템과 상보적
- [x] 처음 30초 안에 효과를 느낄 수 있는가? → 보스 등장 연출 + 첫 공격 패턴으로 즉시 체감

## 2. 설계 결정

### 결정 1: 보스 구현 방식

| 선택지 | 장점 | 단점 | 판정 |
|--------|------|------|------|
| **A: 기존 Enemy + BossTag 확장** | 기존 collision/movement/render 재활용, 코드 중복 최소 | 기존 시스템에 분기문 추가됨 | ✅ 채택 |
| B: 완전 별도 Boss 시스템 | 깔끔한 분리 | 기존 시스템 대부분 중복 구현, 유지보수 부담 | ❌ |

**선택 근거:** ECS의 강점 = 컴포넌트 조합으로 새 엔티티를 만드는 것.  
BossTag 하나 붙이면 기존 Movement, Collision, Render 시스템이 그대로 동작한다.  
보스 전용 로직(페이즈, 이동 패턴)만 BossAISystem에 모으면 깔끔.

### 결정 2: 보스 스테이지 배치

| 선택지 | 장점 | 단점 | 판정 |
|--------|------|------|------|
| A: 매 5스테이지 보스 (5,10,15,20,25) | 세계관 5존과 일치, 예측 가능 | 초반 보스까지 너무 오래 걸림 | ❌ |
| **B: 스테이지 3, 6, 9, 12, 15에 보스** | 적절한 빈도, 3스테이지=~2분마다 보스 | 세계관 5존 매핑이 조금 어색 | ✅ 채택 |
| C: 매 3스테이지 (무한) | 자동 확장 용이 | 보스가 너무 많아 특별함 감소 | ❌ |

**선택 근거 — 세션 타이밍 역산:**

실제 코드 수치(STAGE_DEFS, spawnDelay, clearDuration) 기반 스테이지별 소요시간:

| Stage | 웨이브 | 스테이지 합계 | 누적 시간 |
|:---:|:---:|:---:|:---:|
| 1 | 3 | ~24초 | **0:24** |
| 2 | 4 | ~36초 | **1:00** |
| **3 (보스)** | - | 보스전 30~45초 | **~1:40** |
| 4 | 5 | ~50초 | **2:30** |
| 5 | 5 | ~55초 | **3:25** |
| **6 (보스)** | - | 보스전 35~50초 | **~4:10** |
| ... | | | |
| **15 (보스)** | - | 보스전 50~65초 | **~13:00** |

**왜 5스테이지마다(5,10,15)가 아닌가?**
- 첫 보스가 ~2분 30초에 등장 → 모바일 세션에서 너무 느림
- planning-guide의 "처음 30초 안에 효과를 느낄 수 있는가?" 원칙 위반
- 통학/출퇴근 5분 안에 보스를 한 번은 봐야 "다음 판" 동기가 생김

**왜 3스테이지마다인가?**
- 첫 보스 ~1분 40초 → 모바일 세션에서 "한 판에 보스 1회 이상 조우" 보장
- 3스테이지 = 웨이브 10~15개 처리 → "충분히 준비됐다"는 성취감 + 도전 흥분
- 보스 사이 2스테이지(일반) = **긴장↔이완 파형** 형성:

```
긴장감
  ▲
  │    ╱╲         ╱╲             ╱╲
  │  ╱    ╲     ╱    ╲         ╱    ╲
  │╱  일반  ╲╱   일반  ╲  ... ╱  일반  ╲
  │  1  2  [3보스]  4  5  [6보스]  ...  [15보스]
  └──────────────────────────────────────▶ 시간
```

보스(정점) → 일반(이완) → 일반(고조) → 보스(정점) = Vampire Survivors가 증명한 "적절한 빈도의 climax"

**보스 ↔ 세계관 매핑:**

| 보스 스테이지 | 보스 | 존 | 난이도 역할 |
|:---:|--------|------|------------|
| 3 | **NULL** | Origin (시작) | 튜토리얼 보스. 회피 기본기 검증 |
| 6 | **STACK** | Exploration | 난이도 벽. 복합 패턴 첫 등장 |
| 9 | **HEAP** | Combat Zone | 카오스. 예측 불가 패턴 |
| 12 | **RECURSION** | Danger Zone | 분열 패턴. 미니보스 소환 |
| 15 | **OVERFLOW** | Final Chamber | 최종 보스. 전 패턴 총집합 |

### 결정 3: 보스 페이즈 시스템

| 선택지 | 장점 | 단점 | 판정 |
|--------|------|------|------|
| A: HP 기반 페이즈 | 직관적, 진행도 명확 | 일정한 리듬 |  |
| B: 시간 기반 페이즈 | 예측 가능한 페이싱 | HP와 무관하면 어색 |  |
| **C: 혼합 (HP 기반 전환 + 시간 기반 패턴 순환)** | "얼마나 깎았나" + "다음 뭐 오지" 이중 긴장 | 구현 약간 복잡 | ✅ 채택 |

**동작 방식:**
1. 페이즈(Phase)는 HP 비율로 전환 (예: 66%, 33%)
2. 각 페이즈 내에서 공격 패턴은 시간 순환 (예: 3초 circle → 2초 대기 → 3초 aimed → ...)
3. 페이즈 전환 시 탄막 소거 + 무적 + 새 패턴 예고 (상세: 결정 5 참조)

### 결정 4: 보스 등장 방식

| 선택지 | 장점 | 단점 | 판정 |
|--------|------|------|------|
| **A: 보스 전용 스테이지** | 깔끔, 집중도 높음 | 스테이지 수 증가 | ✅ 채택 |
| B: 웨이브 클리어 후 보스 등장 | 자연스러운 흐름 | 피로 누적 후 보스 = 불공정 |

**선택 근거:** 보스 전용 스테이지 = STAGE_DEFS에 `isBossStage = true` 플래그.  
웨이브 스폰 없이 보스만 등장. 보스 처치 = 스테이지 클리어.  
보스전 직전 레벨업 기회 제공 → 빌드 정리 후 도전.

### 결정 5: 페이즈 전환 시 탄막 처리

| 선택지 | 설명 | 판정 |
|--------|------|:---:|
| **A: 전소거** | 모든 적 탄막 즉시 제거 | MVP 채택 |
| B: 유지 (하드코어) | 탄막 그대로 두고 새 패턴 추가 | ❌ |
| **C: 연출 소거** | 탄막이 느려지며 0.5초 페이드아웃 | 향후 업그레이드 |

**재미 4기둥 기준 비교:**

| 기둥 | A: 전소거 | B: 유지 | C: 연출 소거 |
|------|----------|--------|-------------|
| **긴장감** | 순간 해소 → 다시 축적 (파형) | 끊임없는 압박 (직선) | 잠깐 해소 → 재축적 |
| **성취감** | "한 고비 넘겼다!" 명확 | 달성감 불분명 | "넘기는 중..." 과정 체감 |
| **놀라움** | 탄막 소멸 + 새 패턴 = 뚜렷한 전환 | 패턴 겹침 = 혼돈미 | 비주얼 연출 자체가 놀라움 |
| **몰입감** | 브레이크 → 재진입 리듬 | 쉴 틈 없는 극한 집중 | 자연스러운 흐름 유지 |

**레퍼런스 게임:**

| 게임 | 페이즈 전환 방식 | 결과 |
|------|-----------------|------|
| **Touhou** (동방) | 전소거 + 짧은 무적 | 시리즈 전통. "스펠카드 격파" = 안도감 + 보너스. 탄막슈팅의 교과서 |
| **Cuphead** | 유지 (하드코어) | 극한 난이도가 매력이지만 이탈률도 높음 |
| **Enter the Gungeon** | 연출 소거 | 파티클 전환 연출. 느낌 좋지만 구현 비용 높음 |

**최종 결정: A(전소거)로 MVP → C(연출 소거)로 Phase 3C 업그레이드**

이유:
1. 우리 게임 DNA = "탄막이 예술" → 탄막 소멸도 아름다워야 함 → C가 최종 목표
2. 첫 보스(NULL)는 튜토리얼 → "페이즈가 바뀌었다"를 명확히 알려야 함 → A가 직관적
3. B(유지)는 타겟 유저(캐주얼~미드코어 모바일)에 안 맞음 → Cuphead식 하드코어 = 이탈 유발
4. C는 3C(비주얼 이펙트 Phase)에서 구현 → 지금은 A로 기능 검증

**Phase 전환 동작 (MVP):**
```
1. 보스 0.5초 무적 (데미지 안 받음)
2. 적 탄막 전소거 (MVP: 즉시 / 향후: 0.5초 페이드아웃)
3. 화면 잠깐 플래시 (간단한 피드백)
4. 새 패턴 시작
```

## 3. ECS 분해

### 3.1 새 컴포넌트

#### `bossTag.lua` (03_game/components/)

```lua
-- 마커 컴포넌트 + 보스 메타 정보 + 페이즈 관리
{
    name = "BossTag",
    defaults = {
        bossType = "NULL",       -- "NULL"|"STACK"|"HEAP"|"RECURSION"|"OVERFLOW"
        phase = 1,               -- 현재 페이즈 (1-indexed)
        maxPhase = 2,            -- 총 페이즈 수
        phaseThresholds = {0.5}, -- HP 비율 기준 (phase 1→2: 50%)
        phaseChanged = false,    -- 페이즈 전환 트리거 (1프레임 플래그)

        -- 패턴 순환
        patternIndex = 1,        -- 현재 패턴 인덱스
        patternTimer = 0,        -- 패턴 내 경과 시간
        patterns = {},           -- 페이즈별 패턴 시퀀스 (팩토리에서 세팅)

        -- 연출
        introTimer = 0,          -- 등장 연출 타이머
        introComplete = false,   -- 등장 완료 여부
        defeated = false,        -- 처치 완료 플래그
    }
}
```

### 3.2 새 시스템

#### `bossSystem.lua` (03_game/systems/)

```
requires: [Transform, EnemyAI, Health, BossTag, BulletEmitter]

매 프레임:
1. 등장 연출 관리 (introTimer → introComplete)
2. 페이즈 체크: Health.hp / Health.maxHp ≤ threshold → 페이즈 전환
3. 페이즈 전환 시: 모든 적 탄막 소거, 1초 무적, 패턴 변경
4. 패턴 순환: patternTimer 기반으로 BulletEmitter 패턴/파라미터 교체
5. 보스 이동 패턴: 페이즈별 이동 로직 (좌우 순찰, 중앙 고정 등)
6. 처치 판정: hp ≤ 0 → defeated = true, 보상 트리거
```

### 3.3 기존 시스템 수정

| 파일 | 변경 내용 |
|------|----------|
| `entityFactory.lua` | `createBoss(world, bossType)` 함수 추가 + `BOSS_TYPES` 프리셋 테이블 |
| `stageManager.lua` | `isBossStage` 감지 → 웨이브 대신 보스 스폰, 보스 처치 = 스테이지 클리어 |
| `bulletEmitterSystem.lua` | 보스 전용 패턴 추가 (boss_circle, boss_spiral, boss_aimed, boss_wave) |
| `enemyCollisionSystem.lua` | BossTag 엔티티 처치 시 대량 XP + 보스 처치 이벤트 발행 |
| `topHud.lua` | 보스 HP바 렌더링 (화면 상단, 보스명 표시) |
| `bulletPool.lua` | 보스 탄막 전소 함수 `clearLayer(layer)` 추가 |

### 3.4 UI 변경

- **보스 HP바**: 화면 상단 중앙, 보스명 + HP 게이지
- **스테이지 표시**: "BOSS: NULL" 형태로 보스 스테이지 구분
- **등장 연출**: 간단한 텍스트 + 화면 페이드 (Phase 3C에서 고급 연출 추가)

### 3.5 성능 고려사항

- 보스는 1개만 존재 → 풀링 불필요
- 보스 탄막은 기존 bulletPool 사용 (2000개 충분)
- 페이즈 전환 시 탄막 소거 = bulletPool에 적 탄막 일괄 비활성화 필요
- 보스 패턴 순환 = 테이블 lookup만, 매 프레임 생성 없음

## 4. 보스별 상세 설계

### 4.1 NULL — 튜토리얼 보스 (Stage 3)

> 데이터스피어의 빈 참조. 단순하지만 위협적인 존재.

| 파라미터 | 값 | 비고 |
|----------|-----|------|
| HP | 50 | 기본 basic(HP 3)의 ~17배. 처치 목표 30~45초 |
| 페이즈 | 2 | 100%→50%, 50%→0% |

**HP 50 산출 근거 (DPS 역산):**

플레이어 기본 DPS = fireRate(4) × damage(1) × count(1) = **4 DPS**

Stage 3 도달 시점 플레이어 레벨 추정:
- Stage 1~2 적 처치 XP: basic(2) × ~15마리 + aimed(3) × ~5마리 ≈ 45 XP
- 레벨 2: 10 XP, 레벨 3: 14 XP → **레벨 3 진입, 레벨 4 직전**
- 무기 레벨업 1회 (fireRate ×1.2) 보수적 추정 → **DPS ~4.8**

| HP 후보 | 순수 처치 시간 | 실제 (회피 포함 ×3~4) | 판정 |
|:---:|:---:|:---:|:---:|
| 30 | 6초 | 15~20초 | 너무 쉬움. "이게 보스야?" → 성취감 없음 |
| **50** | **10초** | **30~45초** | **"아슬아슬하게 이겼다" = 긴장감 + 성취감** |
| 75 | 16초 | 50~70초 | 첫 보스에 1분+ → 초보 이탈 위험 |

기존 적 대비 임팩트: basic HP 3 → NULL HP 50 = **16.7배** → "확실히 다른 존재"

> **Feel Before Balance** — 50은 시작점. 플레이테스트에서 조절 (±20% = 40~60 범위).
| 히트박스 반지름 | 0.8 | 일반 적(0.3)의 ~2.7배 |
| 색상 | {0.6, 0.6, 0.6} | 회색 (null = 비어있음) |
| XP 보상 | 50 | 레벨업 1~2회분 |
| 이동 | 상단 좌우 순찰 | y = 상단 1/3 고정, x 왕복 |

**Phase 1 (HP 100%~50%) — "학습"**
| 순서 | 패턴 | 지속 | 설명 |
|:---:|--------|:---:|------|
| 1 | circle (느림) | 3초 | 8발 원형, bulletSpeed 2.0 |
| 2 | 대기 | 2초 | 공격 없음 (회복/위치 조정 기회) |
| 3 | circle (보통) | 3초 | 12발 원형, bulletSpeed 2.5 |
| 4 | 대기 | 2초 | |

**Phase 2 (HP 50%~0%) — "본격"**
| 순서 | 패턴 | 지속 | 설명 |
|:---:|--------|:---:|------|
| 1 | circle + aimed | 3초 | 8발 원형 + aimed 3발 동시 |
| 2 | 대기 | 1.5초 | 대기 시간 단축 |
| 3 | spiral (느림) | 4초 | 2팔 나선, turnRate 1.0 |
| 4 | 대기 | 1.5초 | |

### 4.2 STACK — 레이어 보스 (Stage 6)

> 오염된 스택 데이터. 겹겹이 쌓인 방어를 돌파해야 한다.

| 파라미터 | 값 |
|----------|-----|
| HP | 100 |
| 페이즈 | 3 (66%, 33%) |
| 히트박스 반지름 | 1.0 |
| 색상 | {0.2, 0.8, 0.2} | 
| XP 보상 | 120 |
| 이동 | 중앙 상하 순찰 |

**Phase 1** — 동심원 탄막 (느림, 큰 간격)  
**Phase 2** — 동심원 + aimed (간격 좁아짐)  
**Phase 3** — 3겹 동심원 동시 발사 + 미니언 2마리 소환

### 4.3 HEAP — 카오스 보스 (Stage 9)

> 무작위 메모리 블록. 예측 불가한 공격.

| 파라미터 | 값 |
|----------|-----|
| HP | 160 |
| 페이즈 | 3 (66%, 33%) |
| 히트박스 반지름 | 0.9 |
| 색상 | {0.9, 0.3, 0.1} |
| XP 보상 | 200 |
| 이동 | 랜덤 텔레포트 (5초마다 새 위치) |

**Phase 1** — 랜덤 방향 burst (매번 다른 각도)  
**Phase 2** — burst + wave (불규칙 간격)  
**Phase 3** — 전방위 산탄 + 랜덤 aimed + 이속 증가

### 4.4 RECURSION — 분열 보스 (Stage 12)

> 자기복제 프로세스. 갈수록 많아진다.

| 파라미터 | 값 |
|----------|-----|
| HP | 200 |
| 페이즈 | 3 (66%, 33%) |
| 히트박스 반지름 | 0.7 |
| 색상 | {0.5, 0.1, 0.9} |
| XP 보상 | 300 |
| 이동 | 플레이어 궤도 (원형, 거리 유지) |

**Phase 1** — spiral (자기 중심, 느림). 미니언 없음.  
**Phase 2** — spiral + 미니언 3마리 즉시 소환, 이후 8초마다 보충 (최대 3)  
**Phase 3** — spiral 가속 + aimed 추가 + 미니언 5마리 즉시 소환, 이후 6초마다 보충 (최대 5)

**미니언 상세:**
- 타입: basic (circle 패턴, bulletCount 4~6)
- HP: 기본의 0.5배 (1발~2발에 처치 가능)
- 소환 위치: 보스 주변 반경 2 유닛에 랜덤 배치
- 페이즈 전환 시 기존 미니언 전소거 → 새 페이즈에서 재소환
- 보스 처치 시 미니언 자동 소멸 (LifeSpan 또는 일괄 제거)

**전략적 딜레마:** 미니언을 잡으면 XP + 잠시 편해짐, 하지만 계속 보충됨 → 보스 집중이 정답

### 4.5 OVERFLOW — 최종 보스 (Stage 15) ✅ 구현 완료

> 데이터스피어 붕괴의 근원. 모든 패턴의 종합. "읽을 수 있는 폭주".

| 파라미터 | 값 |
|----------|-----|
| HP | 300 |
| 페이즈 | 4 (75%, 50%, 25%) |
| 히트박스 반지름 | 1.2 |
| 색상 | 빨강 → 초록 → 보라 → 흰색 (Phase별 변화) |
| XP 보상 | 500 |
| 이동 | P1-P2 drift → P3 텔레포트(5s) → P4 텔레포트(3s) |
| 미니언 | P1-P2 없음 → P3-P4 max 3마리/8초 |

**Phase 1** — NULL 오마주: circle only, 빨간색, drift. "이거 봤다" 안심감  
**Phase 2** — STACK 오마주: circle + aimed, 초록색, drift. "좀 세지네" 긴장  
**Phase 3** — HEAP+RECURSION 혼합: burst + wave, 보라색, **텔레포트 5초** + 미니언 3마리. "텔포까지?!"  
**Phase 4** — OVERFLOW 폭주: 4종 패턴 빠른 순환 (spiral→circle→aimed→wave), 흰색, **텔레포트 3초** + 미니언 3마리. "넘친다!"

**핵심 구현**: `phaseTeleport` / `phaseColors` 데이터로 Phase 전환 시 이동 모드/색상 자동 전환. bossSystem은 `teleportInterval > 0` 조건으로 분기 — 코드 추가 없이 데이터만으로 동작.

## 5. 보스 클리어 보상 시퀀스

### 5.1 목적

보스 처치 순간에 플레이어가 느껴야 할 3가지:
1. **카타르시스** — "해냈다!" (긴장 해소)
2. **파워 성장** — "나 강해졌다" (다음 도전 자신감)
3. **브레이크 타임** — 잠깐 숨 돌리기 (몰입 리듬 유지)

### 5.2 보상 내역

| 구분 | 보상 | 재미 기여 |
|------|------|----------|
| 즉시 보상 | 대량 XP 오브 폭발 (20~30개) | 놀라움 + 성취감 |
| 즉시 보상 | 탄막 → XP 변환 (향후) | 놀라움 ("위험이 보상으로") |
| 회복 | HP 전체 회복 | 안도감 |
| 회복 | 대쉬 쿨타임 초기화 | 안도감 |
| 회복 | 포커스 에너지 충전 | 안도감 |
| 성장 | XP 수집 → 보장된 레벨업 1~2회 | 성취감 |
| 연출 | 보스 처치 텍스트 | 세계관 몰입 |

### 5.3 시퀀스 타임라인

```
시간  0.0s ──── 0.3s ──── 0.8s ──── 1.5s ──── 2.0s ──── 4.5s ──── 5.5s
      │         │         │         │         │         │         │
      ▼         ▼         ▼         ▼         ▼         ▼         ▼
   [폭발]   [탄→오브]  [HP회복]  [텍스트]  [수집]   [레벨업]  [다음 진입]
```

**1단계: 폭발 (0.0~0.3초) — 카타르시스**

| 연출 | MVP | 풀 버전 (3C) |
|------|-----|------------|
| 히트스톱 | 0.2초 게임 일시정지 | 동일 |
| 화면 플래시 | 0.1초 화이트아웃 | + 원형 파동 확산 + 화면 흔들림 |

> 히트스톱은 격투게임의 기본. 0.2초가 "방금 대단한 걸 했다"는 느낌을 만들어준다.

**2단계: 탄막 → XP 변환 (0.3~0.8초) — 놀라움**

| 연출 | MVP | 풀 버전 (3C) |
|------|-----|------------|
| 탄막 제거 | 즉시 제거 + 보스에서 XP 폭발 | **탄막이 제자리에서 XP 오브로 변환** |

풀 버전 핵심: 화면의 빨간 탄막이 → 초록 오브로 색이 바뀌며 멈춤.  
**"위험이 보상으로 바뀌는 순간"** = 게임 컨셉 "탄막이 예술"의 극치.  
탄막을 많이 남기고 보스를 잡을수록 보상이 늦 → 전략적 선택 유도.

**3단계: 회복 (0.8~1.5초) — 안도감**

| 연출 | MVP | 풀 버전 (3C) |
|------|-----|------------|
| HP 회복 | 즉시 전체 회복 | 1칸씩 차오르는 애니메이션 |
| 대쉬/포커스 | 즉시 충전 | 게이지 충전 이펙트 |

> 풀 버전에서는 한 번에 확 차는 게 아니라 **차오르는 과정**을 보여주면 "보살핌 받는 느낌".  
> Hades에서 방 클리어 후 보상 줍는 쯨감.

**4단계: 처치 텍스트 (1.5~2.0초) — 세계관 몰입**

화면 중앙 텍스트 표시:

| 보스 | 텍스트 | 의미 |
|------|---------|------|
| NULL | **"NULL — PURIFIED"** | 정화됨 |
| STACK | **"STACK — COLLAPSED"** | 붕괴됨 |
| HEAP | **"HEAP — DEFRAGMENTED"** | 정리됨 |
| RECURSION | **"RECURSION — TERMINATED"** | 종료됨 |
| OVERFLOW | **"OVERFLOW — CONTAINED"** | 봉인됨 |

하단 서브텍스트: "Datasphere Integrity: N%" (진행도 표현)

| 연출 | MVP | 풀 버전 (3C) |
|------|-----|------------|
| 텍스트 | "BOSS CLEAR" 단순 표시 | 보스명 + 키워드 + 글리치 이펙트 |

**5단계: XP 수집 (2.0~4.5초) — 성장**

- 변환된 오브 + 보스 오브가 자석처럼 플레이어에게 빨려옴
- 흡수 속도 점점 가속 (느림 → 빠름) — 쾌감 곡선
- 수집 중 레벨업 트리거 → 레벨업 UI 자연스럽게 등장

**6단계: 다음 스테이지 진입 (4.5~5.5초)**

- 레벨업 선택 완료 후 1초 여유
- "Stage N+1" 텍스트 표시
- 게임 재개

### 5.4 감정 곡선

```
감정 강도
  ▲
  │  ★ 폭발!           ★ 변환!
  │  │                  │        ◆ 텍스트
  │  │   ▓▓▓           │   ▓▓▓  │    ○ 수집~
  │  │   ▓▓▓▓▓         │   ▓▓▓▓▓│    ○○○○
  │──┴───────┴─────────┴────────┴──────────▶ 시간
  0.0   0.3   0.8      1.5  2.0       4.5
  카타르시스  놀라움    안도감  몰입    성장
```

| 보스 | 보스 XP 오브 | 탄막→오브 (향후) | 총 XP 추정 | 보장 레벨업 |

| OVERFLOW | 500 (50개×10) | ~50~80 | 550~580 | 3회 |

> 탄막→오브 변환은 3C 풀버전에서 구현. MVP에서는 보스 XP 오브만 드롭.

## 7. 밸런스 프레임워크

### 보스 처치 시간 목표

**실제 코드 기반 DPS 계산:**

기본 무기: fireRate=4, damage=1, count=1 → **기본 DPS = 4.0**

레벨업 무기 옵션: fireRate ×1.2, damage +1, count +1, speed ×1.25, range ×1.2  
XP per level: 10 → 14 → 19.6 → 27.4 (xpGrowth ×1.4)

| 보스 | Stage | 예상 레벨 | 추정 DPS | HP | 순수 처치 | 실제 (회피 ×3~4) |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| NULL | 3 | 2~3 | ~4.8 | 50 | 10초 | **30~45초** |
| STACK | 6 | 5~6 | ~7.2 | 100 | 14초 | **35~50초** |
| HEAP | 9 | 7~8 | ~10 | 160 | 16초 | **40~55초** |
| RECURSION | 12 | 9~10 | ~14 | 200 | 14초 | **40~55초** |
| OVERFLOW | 15 | 11~12 | ~18 | 300 | 17초 | **50~65초** |

> DPS 추정은 보수적 (무기 옵션 ~50% 선택 가정, bulletCount +1이 DPS에 가장 큰 영향).  
> **Feel Before Balance** — 수치는 시작점. 플레이테스트에서 HP ±20% 범위로 조정.

### 레드라인 체크

- [x] 즉사 금지 — 보스 탄막이 1초 내 화면 가득 안 됨 (모든 패턴에 gap 존재)
- [x] 최소 2가지 회피 경로 — circle은 gap 사이로, aimed는 횡이동으로 회피
- [x] 무적 빌드 불가 — 대쉬 쿨타임 3초, 포커스 에너지 제한
- [x] 레벨업 선택지 함정 없음 — 보스전 영향 없음 (기존 시스템)
- [x] 대기 구간 존재 — 모든 페이즈에 1.5~2초 휴식 포함

### 난이도 스케일링

보스는 고정 스탯 (스테이지 난이도 스케일링 미적용).  
이유: 보스별 수동 디자인 = 체감 품질 우선.  
단, Stage 6+ 자동 구간에서 반복 출현 시 스케일링 고려 (Phase 4).

| ⚠️ 보스 패턴 너무 단조 | 패턴 시퀀스 테이블로 교체 용이. 나중에 추가 |
| ⚠️ bulletPool 2000개 부족? | 보스 탄막은 기존 pool 공유. circle 16발 × 3초 = ~48 동시 → 충분 |
