# 02. 컨셉 설계 (Concept Design)

> Vector Swarm: Zero-Art Roguelite — 모바일 탄막 액션 로그라이트

---

## ★ 재미 철학 (Fun Philosophy)

> **"재미는 내가 추구해야 할 모든 것이다."**

이 게임의 모든 설계, 모든 시스템, 모든 결정은 단 하나의 질문에서 시작한다:

> **"이것은 플레이어에게 재미있는가?"**

재미가 아닌 것은 만들지 않는다. 기술적으로 뛰어나더라도, 시각적으로 아름답더라도, 플레이어가 재미를 느끼지 못하면 실패다. 이 원칙은 개발 전 과정에서 최우선으로 적용된다.

### 재미의 4가지 기둥

```
              ┌──────────────┐
              │  재미 (Fun)   │
              └──────┬───────┘
       ┌─────────┬───┴───┬─────────┐
       ▼         ▼       ▼         ▼
  ┌─────────┐ ┌───────┐ ┌──────┐ ┌────────┐
  │ 긴장감   │  │ 성취감 │ │ 놀라움 │ │ 몰입감  │
  │ Tension │ │Mastery│ │Wonder│ │  Flow  │
  └────┬────┘ └───┬───┘ └──┬───┘ └───┬────┘
       │          │        │         │
  탄막 회피   빌드 완성  패턴 발견  "한 판 더"
```

| 기둥 | 정의 | 이 게임에서의 표현 | 실패 시 증상 |
|------|------|-------------------|-------------|
| **긴장감 (Tension)** | "죽을 수도 있다"는 위험과 회피의 쾌감 | 탄막 사이를 빠져나가는 순간 | 너무 쉽거나, 너무 어려움 |
| **성취감 (Mastery)** | "내가 성장했다"는 실감 | 빌드 완성, 보스 격파, 기록 갱신 | 성장 실감 없음, 보상 부족 |
| **놀라움 (Wonder)** | "이런 것도 되네!"라는 발견 | 새로운 탄막 패턴, 스킬 시너지, 벡터 연출 | 예측 가능, 반복 단조로움 |
| **몰입감 (Flow)** | 시간을 잊고 빠져드는 상태 | 짧은 세션인데 "한 판만 더", 음악+리듬+액션 일체감 | 끊김, 대기시간, 산만함 |

### 재미 검증 체크리스트

모든 기능/시스템 구현 시 아래 질문을 반드시 거친다:

- [ ] 이 기능이 4가지 기둥 중 최소 1가지에 기여하는가?
- [ ] 플레이어가 이 기능을 통해 **즉각적 피드백**을 받는가?
- [ ] 이 기능이 없으면 게임이 **덜 재미있어지는가**?
- [ ] 이 기능이 다른 재미 요소를 **방해하지 않는가**?
- [ ] 처음 30초 안에 재미를 느낄 수 있는가?

> 하나라도 "아니오"면, 설계를 재검토한다.

## 2.1.5. 유니크 아이덴티티 — 이 게임만의 것

> **"다른 게임에서는 느낄 수 없는, Vector Swarm만의 경험은 무엇인가?"**

기존 게임들이 이미 검증한 재미(탄막 회피, 로그라이트 성장, 미니멀 슈팅 등)는 기반이다.
그 위에 **이 게임만이 줄 수 있는 유니크한 경험 3가지**가 핵심 차별점이다.

---

### ① 탄막 자체가 예술 — 위험과 아름다움의 동시 경험

| 기존 탄막 게임 | Vector Swarm |
|--------------|-------------|
| 탄환 = 위협. 피해야 할 것 | 탄환 = 위협 **+ 작품**. 피하면서 감상하는 것 |
| 탄막은 스트레스 | 탄막은 스트레스 **+ 감동** |

- 수학 함수로 생성된 탄막 패턴은 기하학적으로 아름답다
- **"죽고 싶지 않지만 이 패턴은 아름답다"** — 이 모순된 감정이 핵심
- 아름다운 것일수록 위험하다 → 긴장감과 놀라움이 동시에 극대화
- 플레이어가 "이 탄막 스크린샷 찍고 싶다"고 느끼는 순간 = 성공

**재미 기둥:** 긴장감(위험) + 놀라움(아름다움)의 **공존**

---

### ② Zero-Art — 코드가 곧 아트, 수학이 곧 세계

| 기존 게임 | Vector Swarm |
|----------|-------------|
| 도트, 스프라이트, 3D모델 필요 | 수학 함수 = 세계의 전부 |
| 아트 스타일은 "선택" | Zero-Art는 "정체성" |
| 아트 리소스 = 비용 | 코드 = 무한한 비주얼 |

- sin, cos, 프랙탈, 나선 — 이것이 적이고, 배경이고, 이펙트
- 이 세계에 **"그림"이란 존재하지 않는다**. 오직 벡터, 빛, 방정식
- 제약이 아니라 **세계관 자체** (데이터스피어의 모든 것은 벡터)
- 수학을 시각화하는 즐거움 → 개발자도 플레이어도 "만들어지는 과정"이 재미

**재미 기둥:** 놀라움 (이런 것도 수학으로 되네!) + 몰입감 (일관된 세계관)

---

### ③ 탄막을 읽는 재미 — 반사신경이 아닌 지적 쾌감

| 기존 탄막 게임 | Vector Swarm |
|--------------|-------------|
| 반사신경으로 피한다 | **관찰 → 법칙 발견 → 예측 회피** |
| 패턴 암기가 핵심 | 수학적 규칙 이해가 핵심 |
| 못 피하면 "반응이 느렸다" | 못 피하면 "아직 패턴을 못 읽었다" |

- 수학 기반 패턴 → 플레이어가 **법칙을 발견**하는 재미
- "아, 이건 나선형이니까 안쪽으로!" — 관찰 → 이해 → 회피 → 성취
- **지적 쾌감**: "내가 이 패턴을 읽었다"는 느낌 = 단순 반사신경과 다른 만족
- 같은 보스라도 패턴을 이해한 후 → "아, 이제 보인다!" 순간의 쾌감

**재미 기둥:** 성취감 (패턴 이해) + 긴장감 (이해했지만 실행은 아슬아슬)

---

### 유니크 아이덴티티 요약

```
  Vector Swarm = 탄막(위험) + 예술(아름다움) + 수학(지적 쾌감)

  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
  │  ① 탄막=예술  │  +  │ ② Zero-Art  │  +  │ ③ 패턴 읽기  │
  │ 위험+아름다움 │     │ 코드=세계관  │     │ 지적 쾌감    │
  └─────────────┘     └─────────────┘     └─────────────┘
         ↓                   ↓                   ↓
    "스크린샷 찍고    "이 세계에 그림은    "아, 이제
     싶은 위험"        존재하지 않는다"     보인다!"
```

## 2.2. 핵심 재미 심층 분석 (Core Fun Deep Dive)

### 2.2.1. 재미 요소 총괄표

| 순위 | 핵심 재미 | 재미 기둥 | 설명 | 이것이 없으면? |
|------|----------|----------|------|---------------|
| 1 | **탄막 회피의 쾌감** | 긴장감 | 수백 개의 탄환 사이를 실 한 올 차이로 빠져나가는 순간 | 그냥 쏘고 맞는 평범한 슈팅 |
| 2 | **빌드 실험** | 성취감+놀라움 | 매 런마다 다른 무기/스킬 조합으로 새로운 전략 발견 | 매번 같은 플레이, 리플레이 가치 없음 |
| 3 | **시각적 아름다움** | 놀라움 | 탄막 자체가 예술, 스크린샷을 찍고 싶은 장면 | 그냥 위험할 뿐인 탄환 |
| 4 | **성장과 반복** | 성취감 | 짧은 런을 반복할수록 더 강해지는 실감 | 반복이 지루함이 됨 |
| 5 | **"한 판만 더"** | 몰입감 | 5분 세션인데 30분이 지남, 손에서 놓을 수 없음 | 한 판 하고 끝, 다시 켜지 않음 |

### 2.2.2. 재미 요소별 상세 분석

#### 🎯 재미 #1: 탄막 회피의 쾌감 (긴장감)

**왜 재미있나?**
- 인간은 "위험 → 회피 → 생존"의 사이클에서 도파민을 느낀다
- 탄막 사이의 좁은 틈을 통과하는 순간 = 아드레날린 + 성취감
- 동방 시리즈, Enter the Gungeon 등이 증명한 검증된 재미

**구현 핵심:**
- 탄막은 반드시 **"읽을 수 있어야"** 한다 (색상, 속도, 패턴의 예측 가능성)
- **아슬아슬한 회피(Graze)** 시 보상(점수 보너스, 이펙트)
- 판정은 보이는 것보다 **작게** (플레이어에게 유리하게)
- 포커스 모드로 **난이도 자동 조절** (초보자도 접근 가능)

**검증 근거:** Touhou, Ikaruga, Enter the Gungeon, Nuclear Throne

---

#### 🧪 재미 #2: 빌드 실험 (성취감 + 놀라움)

**왜 재미있나?**
- "이번에는 이걸 골라볼까?"라는 **선택의 재미**
- 예상치 못한 시너지 발견 = 뇌에서 **"유레카" 반응**
- 똑같은 게임이 매번 다른 경험 → 무한 리플레이 가치

**구현 핵심:**
- 레벨업 시 **3택 1** — 선택의 의미가 있어야 함 (모두 매력적이어야)
- 스킬/무기 간 **시너지 조합** — 단순 합산이 아닌 곱셈 효과
- "이건 말도 안 되는 조합인데!" → **오버파워(OP) 빌드도 허용**
- 런 초반에 방향이 결정되고, 후반에 완성되는 **성장 곡선**

**검증 근거:** Vampire Survivors, Slay the Spire 2, Hades, Risk of Rain 2

---

#### ✨ 재미 #3: 시각적 아름다움 (놀라움)

**왜 재미있나?**
- 눈이 즐거우면 뇌가 즐겁다 (시각 피드백 = 즉각 보상)
- 탄막이 "아름다울수록" 회피가 더 긴장감 있다 (위험하지만 예쁜 것)
- 스크린샷/영상 공유 → 바이럴, 입소문

**구현 핵심:**
- 탄막 패턴 자체가 **기하학적 예술** (나선, 만다라, 프랙탈)
- 적 처치 시 **파티클 폭발** — 쾌감의 시각화
- 배경이 **살아 움직이는** 수학적 시각화 (sin/cos 파동, 그리드 펄스)
- 화면 전체가 하나의 작품처럼 보이는 순간 만들기

**검증 근거:** Geometry Wars, Rez Infinite, Super Hexagon, Tetris Effect

---

#### 📈 재미 #4: 성장과 반복 (성취감)

**왜 재미있나?**
- "이전보다 더 멀리 갔다" → **진전의 쾌감**
- 영구 강화 해금 → "다음에는 이걸로 해봐야지" → 자연스러운 리플레이
- 도전 과제 → 목표 부여 → 달성 시 보상 쾌감

**구현 핵심:**
- **런 내 성장:** 매 레벨업이 체감될 것 (숫자가 아닌 플레이 경험의 변화)
- **런 외 성장:** 영구 강화가 다음 런에 즉시 영향을 줄 것
- **해금의 기쁨:** 새로운 무기/캐릭터 해금 시 강한 연출과 시운전 기회
- 성장이 "내가 잘해서"라고 느끼게 할 것 (Pay-to-Win 아님)

**검증 근거:** Hades, Dead Cells, Rogue Legacy, Archero

---

#### 🔄 재미 #5: "한 판만 더" (몰입감/Flow)

**왜 재미있나?**
- 심리학의 **Flow 이론**: 도전과 능력이 균형을 이루면 시간을 잊는다
- 짧은 세션 = 부담 없는 시작, 빠른 보상 → 반복 유도
- "이번엔 될 것 같은데..." → 끝없는 재도전

**구현 핵심:**
- 런 시작까지 **3초 이내** (로딩/메뉴 최소화)
- 사망 → 결과 → 재시작까지 **탭 2번 이내**
- 사망 원인이 **명확히 보여야** 함 ("아 그때 그걸 피했어야...")
- 난이도 곡선: 초반 쉬움 → 중반 도전적 → 후반 극한 (자연스러운 오르막)
- **음악과 액션의 싱크** — BPM에 맞춘 적 스폰, 탄막 리듬

**검증 근거:** Flappy Bird(즉사+재시작), Vampire Survivors(시간 녹는 경험), Tetris(Flow의 교과서)

### 2.2.3. 반(反)재미 요소 — 절대 하지 말 것

| 반재미 요소 | 왜 재미를 죽이는가 | 대응 |
|-----------|-------------------|------|
| 긴 로딩/대기 | Flow가 끊긴다 | 즉시 시작, 비동기 로딩 |
| 불공정한 죽음 | "내 잘못이 아닌데" → 분노 | 모든 탄막은 읽을 수 있어야 함 |
| 의미 없는 반복 | 성장 실감 없는 그라인딩 | 매 런이 의미 있는 선택의 연속 |
| 과도한 UI/광고 | 몰입 파괴 | 최소 UI, 인게임 방해 없음 |
| Pay-to-Win | "돈 내면 강해지네" → 성취감 소멸 | 절대 불가. 과금은 꾸미기만 |
| 복잡한 조작 | 모바일에서 조작 스트레스 | 드래그+자동공격, 최대 2~3 제스처 |

## 2.3. 게임 루프 (Core Loop)

```
┌──────────────────────────────────────────────────────┐
│                    메타 루프                           │
│  ┌──────────────────────────────────────────────┐     │
│  │              싱글 런 (5~15분)                  │     │
│  │                                               │     │
│  │  시작 → 스테이지 진행 → 탄막 회피 + 적 처치    │     │
│  │    ↓                                          │     │
│  │  레벨업 → 스킬/무기 선택 (3택 1)               │     │
│  │    ↓                                          │     │
│  │  보스전 → 다음 스테이지 or 사망                │     │
│  └──────────────────────┬───────────────────────┘     │
│                         ↓                             │
│  런 종료 → 재화 획득 → 영구 강화 → 새로운 런 시작    │
└──────────────────────────────────────────────────────┘
```

### 2.3.1. 싱글 런 구조
1. **시작:** 기본 무기 1종으로 출발
2. **스테이지 진행:** 웨이브 기반, 점점 빨라지는 탄막 패턴
3. **레벨업:** 경험치 획득 → 3택 1 스킬/무기 강화 선택
4. **보스전:** 스테이지 끝마다 고유 탄막 패턴의 보스 등장
5. **사망 or 클리어:** 런 종료, 획득 재화로 메타 성장

### 2.3.2. 메타 루프 (영구 성장)
- **재화:** 런 종료 시 "Data Fragment" 획득
- **영구 강화 트리:** 체력, 이동속도, 스킬 슬롯, 신규 무기 해금
- **도전 과제:** 특정 조건 달성 시 보상 (새로운 시작 무기, 캐릭터 등)

## 2.4. 조작 설계 (Controls)

### 2.4.1. 모바일 (Primary)
| 조작 | 입력 | 설명 |
|------|------|------|
| 이동 | 드래그 | 플레이어가 손가락을 따라 이동 |
| 대쉬 | 빠른 스와이프 | 짧은 무적 시간 + 순간이동 (쿨타임 3초) |
| 포커스 모드 | 롱프레스 | 이동 속도 감소 + 판정 축소 + 시간 느려짐 (에너지 소모) |
| 공격 | 자동 발사 | 가장 가까운 적을 자동 조준·발사 |

### 2.4.2. PC (프로토타입)
| 조작 | 입력 | 설명 |
|------|------|------|
| 이동 | 마우스 드래그 / WASD | 마우스 입력 → 터치 시뮬레이션 |
| 대쉬 | 우클릭 / Shift | 순간이동 |
| 포커스 모드 | 좌클릭 홀드 / Space | 포커스 |
| 디버그 | F1~F3 | 디버그 정보 토글 |

## 2.5. 주요 시스템 개요

### 2.5.1. 탄막 시스템 (Bullet System)
- 수학 함수 기반 프로시저럴 탄막 패턴 생성
- sin/cos 조합, 나선형, 방사형, 조준탄, 랜덤 패턴 등
- 오브젝트 풀링으로 300~500개 동시 처리 (60FPS 유지)
- 스테이지 진행에 따라 패턴 복잡도·밀도 증가

### 2.5.2. 적 시스템 (Enemy System)
- **일반 적:** 단순 움직임 + 기본 탄막
- **엘리트 적:** 특수 탄막 패턴 + 높은 체력
- **보스:** 페이즈 전환, 고유 패턴, 공략 포인트
- 적 타입별 행동 트리(Behavior) 분리

### 2.5.3. 성장 시스템 (Progression)
- **런 내 성장:** 레벨업 시 3택 1 (무기/패시브 스킬/능력치)
- **런 외 성장:** Data Fragment로 영구 강화 트리 해금
- **해금 시스템:** 도전 과제 달성 → 신규 무기, 캐릭터, 시작 보너스

### 2.5.4. 월드/맵 설계
- 월드 크기: 120 × 250 world units
- 플레이어 시작점: (0, -100) — 월드 하단부에서 시작, 위로 진행
- 카메라 orthographicSize = 5 (10 world units 가시 높이)
- 웨이브 기반 진행 (맵 탐색 없음, 아레나 형태)
- 스테이지마다 배경 색상/패턴 변화로 시각적 변주

## 2.6. 아트/비주얼 방향

### 2.6.1. Zero-Art 원칙
- 외부 이미지 에셋 없음 (모든 그래픽은 코드로 생성)
- 벡터 도형: 선, 원, 삼각형, 다각형
- 네온 발광 효과 (Bloom, Glow)
- 배경: 그리드 라인, 파동 패턴, 수학적 시각화

### 2.6.2. 컬러 팔레트
| 요소 | 색상 | 용도 |
|------|------|------|
| 플레이어 | 시안/화이트 | 항상 눈에 띄도록 |
| 적 | 레드/오렌지 | 위협 요소 강조 |
| 적 탄환 | 마젠타/핫핑크 | 위험 시각화 |
| 플레이어 탄환 | 그린/라임 | 아군 공격 구분 |
| 배경 | 딥네이비/블랙 | 네온 대비 극대화 |
| UI/HUD | 화이트/시안 | 가독성 우선 |

### 2.6.3. 비주얼 연출
- 적 처치 시 파티클 폭발 (벡터 파편)
- 대쉬 시 잔상 효과 (트레일)
- 보스 등장 시 화면 흔들림 + 배경 색 전환
- 레벨업 시 전체 화면 펄스 이펙트
