## 3C.2 대쉬 잔상 (Dash Ghost Trail)

### 개요
대쉬 순간, 출발→도착 사이에 **고스트 이미지**를 남기고 페이드아웃.
"나 방금 멋있는 걸 했다" 느낌을 준다.

### 설계

| 항목 | 결정 | 이유 |
|------|------|------|
| 잔상 개수 | 4개 | 출발↔도착 균등 배치. 너무 적으면 빈약, 많으면 노이즈 |
| 지속시간 | 0.4초 페이드아웃 | 빠른 액션감 유지 |
| 색상 | 플레이어 색 (시안) + 알파 감소 | 일관성. 뒤쪽 잔상이 더 투명 |
| 형태 | 플레이어 본체 (원) 복제 | 단순, 인식 명확 |
| 데이터 | Dash 컴포넌트에 `ghosts` 배열 | ECS 패턴 유지 |

### 구현 계획

**변경 파일:**
1. `dash.lua` — `ghosts = {}` 필드 추가
2. `dashSystem.lua` — 대쉬 실행 시 4개 고스트 위치 생성, 매 프레임 타이머 감소
3. `playerRenderSystem.lua` — 고스트 draw (반투명 원, 시간에 따라 알파 감소)

**고스트 데이터 구조:**
```lua
ghost = {
    x = ..., y = ...,     -- 위치
    alpha = 1.0,          -- 초기 불투명도
    timer = 0.4,          -- 남은 시간
    maxTime = 0.4,        -- 페이드 계산용
    radius = ...,         -- 플레이어 반지름
}
```

**의사 코드:**
```
-- dashSystem: 대쉬 실행 시
local startX, startY = transform.x, transform.y
-- (대쉬 이동 실행)
local endX, endY = transform.x, transform.y
for i = 1, 4 do
    local t = (i - 1) / 3  -- 0, 0.33, 0.67, 1.0
    ghost[i] = lerp(start, end, t)
end

-- dashSystem: 매 프레임
for each ghost: timer -= dt, remove when timer <= 0

-- playerRenderSystem: draw
for each ghost: alpha = timer / maxTime, draw circle
```

## 3C.3 화면 흔들림 (Screen Shake) — 계획

### 개요
보스 등장, 대형 폭발, 피격 시 카메라를 짧게 흔들어 **임팩트** 전달.

### 설계 (예정)

| 항목 | 방향 |
|------|------|
| 트리거 | 보스 등장, 보스 처치, 플레이어 피격 |
| 강도 | 이벤트별 다름 (보스 등장 > 피격) |
| 구현 위치 | camera.lua에 shake 오프셋 추가 |
| 감쇠 | 지수 감소 (exponential decay) |

## 3C.5 Graze 이펙트 (Near-Miss Feedback) — 계획

### 개요
적 탄환이 플레이어 근처를 스쳐갈 때 **시각 피드백** + 보상.
"위험했지만 살았다" → 긴장감 + 성취감.

### 설계 (예정)

| 항목 | 방향 |
|------|------|
| 판정 | 탄환-플레이어 거리 < graze 반경 (충돌 반경의 2~3배) |
| 시각 | 플레이어 주변 스파크/번쩍임 |
| 보상 | 포커스 에너지 회복 or 점수 보너스 |
| 구현 | collisionSystem에 graze 체크 추가 |
